# Route Content Schema

Each route project should keep both human-readable files and structured machine-readable files.

## Project Files

- `project.json`: route metadata and RouteMarket IDs.
- `brief.md`: research brief.
- `sources.json`: collected sources.
- `notes.md`: working notes.
- `claims.json`: verified and uncertain claims.
- `poi.geojson`: points of interest.
- `route_concept.md`: route concept.
- `route.gpx`: GPX track when available.
- `route.geojson`: route geometry when available.
- `route_summary.json`: validated route metrics when available.
- `guide.md`: RouteMarket-ready guide draft.
- `tips.json`: wizard tips for RouteMarket.
- `recommendations.json`: recommended places for RouteMarket.
- `media/manifest.json`: AI/owned/licensed asset manifest.
- `routemarket_payload.json`: prepared draft payload.
- `review_checklist.md`: human review checklist.
- `quality_report.md`: quality and review status.
- `media/license_report.md`: media provenance and permissions.

## Draft Content Structure

- title,
- short intro,
- who this route is for,
- route overview,
- why it is worth doing,
- route description,
- key places,
- practical information,
- safety notes,
- best season,
- gear and preparation,
- variants,
- sources and verification,
- disclaimer.
